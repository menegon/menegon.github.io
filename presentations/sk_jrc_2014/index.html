The RITMARE Suite Software: Objectives

Bottom-up strategy: enable researchers to autonomously contribute to the infrastructure

  • enablement of independent nodes of geo-data provision (inside and outside RITMARE):
    • maps/layers
    • observations
  • semantics-enriched (and -assisted) MD creation:
    • fine-grained Metadata (data-sets, sensors)
    • leveraging controlled vocabularies and RDF-based data structures (FOAF description of the project’s researcher, organisation, ).

Software Components (FLOSS)

  • GeoNode 2.0
    • GeoServer 2.4
    • PyCSW
    • Django web framework
    • PostgreSQL/PostGIS
  • SOS 52°North 4.0.0
    • Embedded EDI Metadata
  • RITMARE Software Suite (formerly StarterKit)

RITMARE Software Suite: additional components

  • SOS Manager Supports editing of SensorML for registering a new sensor and allows upload of observations as tabular data in a text file.
  • SOS Client The interface allows to easily interact with an SOS to retrieve, for each sensor, position, parameter list, offerings, and values.
  • EDI Client The client-side component of the customizable, semantics-aware metadata editing service.
